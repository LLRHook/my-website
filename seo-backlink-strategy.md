# Backlink Strategy — victorivanov.engineer

Goal: Build domain authority and improve search rankings for "Victor Ivanov software engineer" by acquiring high-quality backlinks from relevant sources.

## Immediate Actions (Week 1)

### 1. Google Search Console
- Verify ownership of `victorivanov.engineer`
- Submit sitemap: `https://victorivanov.engineer/sitemap.xml`
- Request indexing for the homepage
- Monitor coverage report weekly

### 2. GitHub Profile Optimization
- Set website URL to `https://victorivanov.engineer` in GitHub profile settings
- Add a professional bio mentioning "Software Engineer" and key technologies
- Pin top repositories that link back to the portfolio in their README
- Ensure each pinned repo README includes a link to `victorivanov.engineer`

### 3. LinkedIn Profile
- Set website field to `https://victorivanov.engineer`
- Add portfolio link in the "Featured" section
- Include link in any posts about projects or technical topics

### 4. X (Twitter) Profile
- Set website URL to `https://victorivanov.engineer`

## Short-Term Actions (Weeks 2–4)

### 5. Developer Directories
Submit profile to high-authority developer directories:
- **dev.to** — Create profile, link to portfolio, cross-post technical content
- **Hashnode** — Create blog, link to portfolio in bio and posts
- **Stack Overflow** — Add website URL to profile (high DA backlink)
- **npm** — If publishing packages, set homepage to portfolio URL
- **Product Hunt** — Create a maker profile with portfolio link
- **Wellfound (AngelList)** — Create developer profile

### 6. Technical Profiles
- **HackerRank / LeetCode** — Add portfolio URL to profiles
- **Kaggle** — If doing any ML/AI work, link portfolio
- **Google Developer Profile** — Create and link

### 7. University Alumni Networks
- **UMBC Alumni Directory** — Update profile with current role and website
- **Georgia Tech Alumni Network** — Same as above
- Alumni directories often have high domain authority

## Medium-Term Actions (Months 1–3)

### 8. Technical Blog Posts
Write 2–3 technical articles and cross-post to:
- **dev.to** (DA ~60)
- **Hashnode** (DA ~70)
- **Medium** (DA ~95)
- Each post should naturally link back to portfolio or specific projects

Suggested topics:
- "Building a Portfolio with Next.js App Router and SEO Best Practices"
- "Structured Data for Developer Portfolios: A Practical Guide"
- "Full-Stack Project Architecture with Spring Boot and React"

### 9. Open Source Contributions
- Contribute to well-known open source projects
- Author bio / contributor profile links back to portfolio
- Open source project READMEs that credit contributors are valuable backlinks

### 10. Guest Posts & Collaboration
- Reach out to tech bloggers for guest post opportunities
- Collaborate on open source tools with other developers
- Co-author technical articles

## Ongoing Actions

### 11. Social Proof & Mentions
- Share projects on relevant subreddits (r/webdev, r/nextjs, r/reactjs)
- Post project showcases on Hacker News (Show HN)
- Engage in technical discussions with portfolio link in bio

### 12. Project-Based Backlinks
- Each new project should have its own README linking to the portfolio
- Deploy projects to custom subdomains that link back to main site
- Submit interesting projects to showcase directories (Made with React, Next.js Showcase)
